# Инструкция по созданию тестового студента

## Для локальной разработки:

1. **Создайте тестового студента с данными:**
   ```bash
   python manage.py create_test_student
   ```

   Это создаст:
   - Пользователя: `test_student` / пароль: `test123456`
   - Группу: CS-101
   - 3 курса (Введение в программирование, Базы данных, Веб-разработка)
   - Расписание на неделю
   - Записи на курсы

2. **Войдите как тестовый студент:**
   - Логин: `test_student`
   - Пароль: `test123456`

3. **Просмотрите данные:**
   - Расписание: `/schedule/`
   - Группы: `/groups/`
   - Курсы: в дашборде

## Для Render (после деплоя):

1. **Откройте Render Shell** (в настройках вашего сервиса)

2. **Выполните команду:**
   ```bash
   python manage.py create_test_student
   ```

3. **Войдите на сайт:**
   - Логин: `test_student`
   - Пароль: `test123456`

## Альтернатива: Создание полных демо-данных

Если нужно больше данных (500 студентов, 20 групп, 30 курсов):

```bash
python manage.py seed_demo --students 500 --groups 20 --courses 30 --seed 42
```

После этого можно зарегистрировать нового студента и выбрать группу при регистрации.

## Просмотр всех групп

На сайте доступна страница со списком всех групп:
- URL: `/groups/`
- Показывает все группы с количеством студентов
- Можно посмотреть расписание каждой группы

## Что видит студент:

1. **Расписание** (`/schedule/`) - показывает расписание студента по его группе
2. **Группы** (`/groups/`) - список всех групп и их расписание
3. **Курсы** - в дашборде показываются курсы, на которые записан студент
4. **Оценки** (`/grades/`) - оценки студента

## Если расписание пустое:

1. Убедитесь, что студент привязан к группе (в профиле)
2. Убедитесь, что для группы создано расписание
3. Проверьте, что ScheduleEntry связан с профилями студентов через ManyToManyField `groups`
